GitHub legfontosabb parancsai

  1. git init
    • A git init parancs lehetőséget ad arra, hogy új repositoryt hozzunk létre. Általában ez az első parancs amit egy GitHub repository létrehozásakor használunk. A parancsban a létrehozandó repository nevét is megadhatjuk.
      Használata: git init / git init *repositoy név*.
  2. git clone
    • A git clone parancs egy local working copy-t lokális másolatot hozlétre a távoli könyvtárből (remote repository). Amikor egy könyvtárat klónozol akkor az abban levő fájlok, kódok autómatikusan letöltődnek a gépedre. Ez a parancs ad egy original location-t mint amilyen a távoli könyvtár , szóval húzhatsz le róla változtatásokat és fel is töltheted a változtatásaidat már hogyha van engedélyed hozzá.
      Használata: git clone *git repo url*
  3. git branch
    • A git branch parancsal egy új ágat (branch) adhatunk amár meglevő águnkhoz, kiirja a már létező ágakat és törölhetünk is ágat.
      Használat: git branch *barnch név* / git branch -a / git branch --list / git branch -d *branch név*
  4. git checkout
    • Ezzel a parancsal át léphetünk egy másik ágra, vagy létrehozunk egyet és átlépünk abba. Ehez szükséges hogy a branch amibe át szeretnénk lépni létezzen a local repoba, illetve a jelenlegu branch-en levő változások commitolva vagy stash-elve kell legyen.
      Használata: git checkout *branch név* / git checkout -b *új branch név*
  5. git add
    • A git add parancs hozzá adja a változtatásokat a staging area-hoz amiben össze hasonlíthatod a lokális verziódat és azt a verziót ami a távoli könyvtárba van
      Használata: git add *file név 1* *file név 2* / git add -A
  6. git commit
    • A git commit parancs elmenti a változtatásokat a lokális könyvtáradba (local repository), ehez még hozzá kell adni egy rövid leírást, hogy milyen változtatások vannak benne.
      Használat: git commit -m 'commit-üzenet'
  7. git push
    • Ez a parancs feltolja (push) a commitált fájlok változtatásait a lokális repobol a távoli repoba , illetve csinál egy elnevezett branch-et a remote repoba ha még nem létezne.
      Használata: git push / git push *remote* *branch*
  8. git pull
    • A git pull le húzza a lokális repoba a legkésőbbi verziót ami a remote szerveren taláható
      Használat: git pull / git pull *remote* / git pull *remote* *branch név*
  9. git merge
    • A git merge parancs össze 'olvasztja' az branch-et a szülő branch-el. A azülő lehet development vagy master branch ez a munkafolyamattól függ. Automatikusan létre fog hozni egy comitot ha nincsen konflikt.
      Használat: git merge *branch név*
  10. git status
    • A git status kiirja az adott helyzetedet az adott repository-ba.
      Használat: git status
